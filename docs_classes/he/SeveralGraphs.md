# תיאור המחלקה SeveralGraphs

## 1. מידע כללי
*   **שם המחלקה:** `SeveralGraphs`
*   **סוג:** Activity
*   **ייעוד:** מחלקה זו מראה כיצד להציג מספר פונקציות ריבועיות (פרבולות) על גרף אחד. היא עוברת בין ארבעה סטים מוגדרים מראש של מקדמים כדי להראות כיצד שינוי של 'a', 'b' או 'c' משפיע על הצורה והמיקום של הפרבולה.
*   **אינטראקציה:** מופעלת מ-`MainActivity`. היא משתמשת ב-`GraphView` משותף להצגת מספר סדרות נתונים (`LineGraphSeries`).

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `a`, `b`, `c` | `int` | מקדמי בסיס לפרבולות | `viewGraph`, `buildGraph` |
| `numPoints` | `int` | מספר הנקודות לחישוב עבור כל גרף | `buildGraph` |
| `i` | `int` | מונה למעקב אחר הגרף הבא שיש לצייר | `viewGraph`, `buildGraph` |
| `graphS` | `GraphView` | רכיב ממשק להצגת הגרפים | `onCreate`, `buildGraph` |
| `dx` | `double` | גודל הצעד לחישוב ציר ה-X | `buildGraph` |
| `colors` | `int[]` | מערך צבעים (שחור, כחול, אדום, מגנטה) לגרפים | `buildGraph` |

## 3. שיטות המחלקה

### שם השיטה: `onCreate`
*   **סוג השיטה:** `protected`
*   **ערך חזרה:** `void`
*   **פרמטרים:** `Bundle savedInstanceState`
*   **מה השיטה עושה:** מגדירה את הפריסה ומאתחלת את ההפניה ל-`graphS`.
*   **מתי נקראת:** בעת יצירת ה-Activity.

### שם השיטה: `viewGraph`
*   **סוג השיטה:** `public`
*   **ערך חזרה:** `void`
*   **פרמטרים:** `View view`
*   **מה השיטה עושה:** 
    1. מקדמת את המונה `i`.
    2. בהתאם לערך של `i`, היא קוראת ל-`buildGraph` עם וריאציות שונות של מקדמי הבסיס (למשל, 'a' שלילי, 'b' שלילי או 'c' שלילי).
*   **מתי נקראת:** כאשר המשתמש לוחץ על כפתור "View Graph".

### שם השיטה: `buildGraph`
*   **סוג השיטה:** `public`
*   **ערך חזרה:** `void`
*   **פרמטרים:** `int a, int b, int c`
*   **מה השיטה עושה:** 
    1. יוצרת סדרת נתונים חדשה (`LineGraphSeries`).
    2. מחשבת $y = ax^2 + bx + c$ עבור $x$ החל מ-20- ומעלה בצעדים של `dx`.
    3. מוסיפה את הנקודות המחושבות לסדרה.
    4. מוסיפה את הסדרה לגרף.
    5. קובעת את צבע הקו בהתאם למונה הנוכחי `i`.
*   **מתי נקראת:** מתוך `viewGraph`.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`:** אתחול וקישור ממשק המשתמש.

## 5. אינטראקציה עם הממשק (UI)
*   **כפתור:** מפעיל הוספת גרף חדש בכל לחיצה.
*   **GraphView:** צובר ומציג את כל הסדרות שנוספו.

## 6. אינטראקציה עם רכיבים אחרים
*   מופעלת באמצעות Intent מ-`MainActivity`.

## 7. לוגיקה כללית של המחלקה
כל לחיצת כפתור "מוסיפה" פרבולה חדשה לתרשים הקיים. היא משתמשת במונה כדי להחליט איזו וריאציה מתמטית להציג בשלב הבא, מה שמאפשר למשתמש להשוות בין ארבעה מצבים שונים של המשוואה הריבועית.

## 8. הסבר מפושט
תארו לעצמכם שיש לכם נייר שקוף מעל רשת. בכל פעם שאתם לוחצים על הכפתור, האפליקציה מציירת עקומה צבעונית חדשה. בלחיצה הרביעית, יש לכם ארבע עקומות שונות על אותו נייר, מה שמקל לראות איך שינוי המספרים בנוסחה מזיז את העקומה.

**אנלוגיה:** זה כמו למדוד ארבע חולצות בצבעים שונים אחת מעל השנייה כדי לראות איזו מהן מתאימה הכי טוב או איך הן שונות.
