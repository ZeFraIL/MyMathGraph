# תיאור המחלקה LinearFunction

## 1. מידע כללי
*   **שם המחלקה:** `LinearFunction`
*   **סוג:** Activity (מסך באפליקציה)
*   **ייעוד:** מחלקה זו אחראית להצגה חזותית של פונקציה ליניארית מהצורה $y = ax + b$. היא מקבלת קלט מהמשתמש עבור המקדמים 'a' ו-'b' ומשרטטת את הקו המתאים על גרף.
*   **אינטראקציה:** היא מופעלת מה-`MainActivity` באמצעות `Intent`. היא משתמשת בספריית `GraphView` כדי לרנדר את השרטוט המתמטי.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `etA1` | `EditText` | שדה קלט למקדם 'a' | `onCreate`, `viewGraph` |
| `etB1` | `EditText` | שדה קלט למקדם 'b' | `onCreate`, `viewGraph` |
| `stA` | `String` | אחסון זמני לטקסט מ-`etA1` | `viewGraph` |
| `stB` | `String` | אחסון זמני לטקסט מ-`etB1` | `viewGraph` |
| `a` | `int` | הערך המספרי של מקדם 'a' | `viewGraph` |
| `b` | `int` | הערך המספרי של מקדם 'b' | `viewGraph` |
| `graphL` | `GraphView` | רכיב ממשק המציג את הגרף | `onCreate`, `viewGraph` |
| `x` | `double` | קואורדינטת ה-x של שורש הפונקציה ($y=0$) | `viewGraph` |
| `xL` | `double` | קואורדינטת ה-x לנקודה השמאלית של הקו | `viewGraph` |
| `xR` | `double` | קואורדינטת ה-x לנקודה הימנית של הקו | `viewGraph` |
| `y` | `double` | קואורדינטת y מחושבת | `viewGraph` |

## 3. שיטות המחלקה

### שם השיטה: `onCreate`
*   **סוג השיטה:** `protected`
*   **ערך חזרה:** `void` (לא מחזירה דבר)
*   **פרמטרים:** `Bundle savedInstanceState` (מכיל את המצב השמור של המסך)
*   **מה השיטה עושה:**
    1.  קוראת לבנאי של מחלקת האב.
    2.  קובעת את הפריסה (Layout) ל-`activity_linear_function`.
    3.  מקשרת בין משתני ה-Java (`etA1`, `etB1`, `graphL`) למקביליהם בקובץ ה-XML באמצעות `findViewById`.
*   **מתי נקראת:** באופן אוטומטי על ידי מערכת אנדרואיד בעת יצירת המסך לראשונה.

### שם השיטה: `viewGraph`
*   **סוג השיטה:** `public`
*   **ערך חזרה:** `void`
*   **פרמטרים:** `View view` (הכפתור שנלחץ)
*   **מה השיטה עושה:**
    1.  קוראת את הטקסט משדות הקלט.
    2.  בודקת אם השדות ריקים (אימות בסיסי).
    3.  ממירה טקסט למספרים שלמים (`Integer.parseInt`).
    4.  מחשבת את שורש הפונקציה: $x = -b / a$.
    5.  יוצרת סדרת נתונים חדשה (`LineGraphSeries`).
    6.  מחשבת שתי נקודות: אחת 5 יחידות שמאלה מהשורש ואחת 5 יחידות ימינה.
    7.  מוסיפה נקודות אלו לסדרה ומוסיפה את הסדרה לגרף.
*   **מתי נקראת:** כאשר המשתמש לוחץ על כפתור "View Graph".
*   **חשוב להבין:**
    *   **באג פוטנציאלי:** אם המשתמש יזין `a = 0`, התוכנית תקרוס עקב חלוקה באפס.
    *   **תכונה:** נעשה שימוש בשתי נקודות בלבד לשרטוט הקו מכיוון שפונקציה ליניארית היא תמיד קו ישר.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`:** נקראת בעת הפעלת ה-Activity. זה המקום שבו אנו מגדירים את הממשק ומוצאים את הרכיבים.

## 5. אינטראקציה עם הממשק (UI)
*   **רכיבים:** `EditText` (למספרים), `Button` (להפעלת החישוב), `GraphView` (להצגת התוצאה).
*   **קישור:** `findViewById` משמש לקישור מזהי XML לאובייקטי Java.
*   **אירועים:** אירוע `onClick` על הכפתור (מוגדר ב-XML באמצעות `android:onClick="viewGraph"`).

## 6. אינטראקציה עם רכיבים אחרים
*   **Intent:** מסך זה נפתח מה-`MainActivity`.
*   **העברת נתונים:** הוא אינו מקבל נתונים ממסכים אחרים; הוא מסתמך על קלט המשתמש.

## 7. לוגיקה כללית של המחלקה
1.  המשתמש מזין 'a' ו-'b'.
2.  המשתמש לוחץ על הכפתור.
3.  האפליקציה מחשבת היכן הקו חוצה את ציר ה-X.
4.  האפליקציה משרטטת קו ישר דרך הנקודה הזו.

## 8. הסבר מפושט
תארו לעצמכם שיש לכם סרגל ואתם רוצים לצייר קו על דף נייר. כדי לעשות זאת, אתם צריכים לפחות שתי נקודות. מחלקה זו מחשבת את שתי הנקודות הללו על סמך המספרים שאתם נותנים לה ומשתמשת ב"סרגל דיגיטלי" (ספריית `GraphView`) כדי לצייר את הקו עבורכם.

**אנלוגיה:** זה כמו מתכון. אתם נותנים את המרכיבים (מספרים 'a' ו-'b'), והיא "מבשלת" שרטוט של קו.
