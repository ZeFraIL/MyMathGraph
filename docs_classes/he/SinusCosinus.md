# תיאור המחלקה SinusCosinus

## 1. מידע כללי
*   **שם המחלקה:** `SinusCosinus`
*   **סוג:** Activity
*   **ייעוד:** מחלקה זו מציגה פונקציות טריגונומטריות: סינוס, קוסינוס וסינוס עם הזזת פאזה. היא מראה כיצד הגלים הללו קשורים זה לזה.
*   **אינטראקציה:** מופעלת מ-`MainActivity`. משתמשת ב-`GraphView` עם מספר סדרות נתונים.

## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `amp` | `int` | אמפליטודה (גובה הגל) | `onCreate` |
| `numPoints` | `int` | רזולוציה (מספר נקודות עבור 720 מעלות) | `onCreate` |
| `graphSiCo` | `GraphView` | מכל הגרף | `onCreate` |
| `y` | `double` | ערך Y מחושב | `onCreate` |
| `f` | `double` | קבוע הזזת פאזה ($\pi/4$) | `onCreate` |
| `series1`, `series2`, `series3` | `LineGraphSeries` | קווים שונים עבור סינוס, קוסינוס וסינוס מוזז | `onCreate` |

## 3. שיטות המחלקה

### שם השיטה: `onCreate`
*   **סוג השיטה:** `protected`
*   **ערך חזרה:** `void`
*   **פרמטרים:** `Bundle savedInstanceState`
*   **מה השיטה עושה:**
    1.  מגדירה את הממשק.
    2.  מחשבת וממלאת את `series1` (סינוס) בצבע שחור.
    3.  מחשבת וממלאת את `series3` (סינוס + הזזת פאזה) בצבע אדום.
    4.  מחשבת וממלאת את `series2` (קוסינוס) בצבע מגנטה.
    5.  מוסיפה את כל הסדרות לגרף.
*   **מתי נקראת:** בעת הפעלת ה-Activity.
*   **חשוב להבין:** הלולאה עוברת מ-0 עד 720, המייצגים מעלות. הנוסחה `i * Math.PI / 180` ממירה מעלות לרדיאנים, כפי שנדרש על ידי `Math.sin()`.

## 4. מחזור חיים (Lifecycle)
*   **`onCreate()`:** כל לוגיקת השרטוט קורה כאן מיד.

## 5. אינטראקציה עם הממשק (UI)
*   מציגה את הגרף ישירות ללא קלט משתמש במחלקה ספציפית זו (פרמטרים קבועים בקוד).

## 6. אינטראקציה עם רכיבים אחרים
*   מעבר Activity סטנדרטי מ-`MainActivity`.

## 7. לוגיקה כללית של המחלקה
היא מחשבת מראש שלושה גלים שונים ומציגה אותם על מערכת צירים אחת כדי להראות את ההבדלים ביניהם (פאזה וצורה).

## 8. הסבר מפושט
תארו לעצמכם שלושה רקדנים הנעים בגלים. אחד מתחיל באפס (סינוס), אחד מתחיל למעלה (קוסינוס), ואחד מתחיל מעט קודם (סינוס מוזז). מחלקה זו מציירת את המסלולים של כל שלושת הרקדנים בבת אחת כדי שתוכלו להשוות ביניהם.
